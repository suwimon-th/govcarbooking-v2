1. บทนำ: โครงสร้างผู้ใช้งาน

ระบบแบ่งผู้ใช้งานเป็น 3 ระดับ:

  1. User: ข้าราชการ/เจ้าหน้าที่ทั่วไป → ทำหน้าที่ "จอง" และ "ดูสถานะ"
  2. Driver: พนักงานขับรถ → ทำหน้าที่ "รับงาน", "ลงบันทึกไมล์", "เบิกน้ำมัน"
  3. Admin: เจ้าหน้าที่ยานพาหนะ → ทำหน้าที่ "บริหารจัดการทั้งหมด"

PART A: สำหรับผู้ใช้งานทั่วไป (User)

A1. การเข้าสู่ระบบ

  • เข้าเว็บไซต์ผ่าน Browser (Chrome, Safari, Edge)
  • กรอก Username และ Password ที่ได้รับจากเจ้าหน้าที่
  • กดปุ่ม "เข้าสู่ระบบ" (สีน้ำเงิน)

หมายเหตุ: กรณีลืมรหัสผ่าน ให้ติดต่อ Admin เพื่อทำการ Reset (ไม่มีปุ่มกดเองเพื่อความปลอดภัย)

A2. หน้าหลักและปฏิทิน (Dashboard)

เมื่อเข้ามาแล้ว จะพบกับ ปฏิทินการใช้รถ

  • Desktop (คอมพิวเตอร์): แสดงตารางเต็มเดือน มีแถบสีคาดในวันที่รถไม่ว่าง
  • Mobile (มือถือ): แสดงปฏิทินย่อ (มีจุดสี) เมื่อแตะที่วันที่ รายการจองจะแสดงด้านล่าง
  • การกรอง: สามารถกดปุ่มลูกศร < > เพื่อดูเดือนถัดไป/ย้อนหลังได้

A3. การการขอใช้รถ (Request Booking)

Flow การจอง:

  1. กดปุ่ม "ขอใช้รถใหม่ (+)"
  2. ระบบจะเปิดแบบฟอร์ม ให้กรอกข้อมูลดังนี้:
    • ผู้ขอ/ตำแหน่ง: ระบบดึงให้อัตโนมัติ (แก้ไขตำแหน่งได้)
    • วันเดินทาง: เลือกวันที่ (ห้ามเลือกย้อนหลัง)
    • เวลา: ระบุเวลาไป-กลับ
    • เลือกรถ: กดเลือกจากรายการรูปรถ (จะมี Pop-up แสดงรถที่ว่างและไม่ว่าง)
    • คนขับ (กรณี OT): หากเลือกเวลานอกราชการ (ก่อน 08:00 หรือหลัง 16:00) ระบบจะบังคับให้เลือกคนขับ เพื่อให้แจ้งเตือน Line
    • สถานที่/ภารกิจ: ระบุให้ชัดเจน
    • ผู้โดยสาร: ระบุจำนวนและชื่อ (เลือกจากรายชื่อเจ้าหน้าที่ หรือพิมพ์ชื่อบุคคลภายนอก)
  3. กด "บันทึก" เมื่อเสร็จสิ้น สถานะจะเป็น "รออนุมัติ (Pending)"

A4. การติดตามสถานะและประวัติ

  • ดูที่หน้าปฏิทิน:
    • สีเหลือง Pending: รอแอดมินดำเนินการ
    • สีเขียว Approved: ได้รับรถแล้ว (คลิกดูรายละเอียดจะเห็นชื่อคนขับและเบอร์โทร)
    • สีแดง Rejected: คำขอถูกปฏิเสธ (ดูเหตุผลในรายละเอียด)

A5. การแจ้งปัญหา (Report Issue)

  • หากพบปัญหาเกี่ยวกับรถ (เช่น แอร์ไม่เย็น, รถเสีย, ความสะอาด)
  • กดปุ่ม "แจ้งปัญหา" (ไอคอนตกใจ ⚠) ที่เมนูช่วยเหลือ หรือในรายละเอียดรถ
  • ระบุรายละเอียดและแนบรูปภาพ (ถ้ามี) เรื่องจะถูกส่งตรงถึง Admin ทันที

PART B: สำหรับพนักงานขับรถ (Driver)

B1. การรับงานและการแจ้งเตือน (Line)

  • การเชื่อมต่อ: พนักงานขับรถต้องทำการ "Link Account" กับ LINE OA ของระบบ (สแกน QR Code ที่ Admin ออกให้)
  • เมื่อมีงานเข้า: ระบบจะเด้งแจ้งเตือนผ่าน LINE บอกรายละเอียด (วันที่, เวลา, สถานที่, ผู้จอง)

B2. การบันทึกเลขไมล์ (Mileage)

เป็นหน้าที่สำคัญที่ต้องทำ ทุกครั้งที่ออกรถ:

  1. ขาไป (Start):
    • เข้าระบบ Driver > เมนู "บันทึกไมล์เริ่ม"
    • เลือกงานที่กำลังจะไป > กรอกเลขไมล์หน้าปัดปัจจุบัน > กดบันทึก
  2. ขากลับ (End):
    • เมื่อส่งผู้โดยสารเสร็จ > เข้าระบบ > เมนู "บันทึกไมล์จบ"
    • กรอกเลขไมล์สิ้นสุด > ระบบจะคำนวณระยะทางรวม (Trip Distance) ให้ทันที

B3. การขอเบิกน้ำมัน (Fuel Request)

  • เมื่อเติมน้ำมัน ให้เข้าเมนู "แจ้งเติมน้ำมัน"
  • ถ่ายรูปใบเสร็จ หรือระบุจำนวนลิตร/ราคา
  • กดส่ง → รายการจะไปโผล่ที่หน้าจอ Admin เพื่อรออนุมัติเบิกจ่าย

PART C: สำหรับเจ้าหน้าที่ดูแลระบบ (Admin)

C1. ภาพรวมและสถิติ (Overview)

  • หน้า Dashboard จะสรุปสถานะรถวันนี้ (ว่างกี่คัน, ใช้งานกี่คัน)
  • กราฟสรุปยอดการใช้รถรายเดือน

C2. การจัดการคำขอ (Requests Management)

หน้านี้คือหัวใจสำคัญของ Admin:

Flow: อนุมัติงาน (Approval)

  1. ไปที่เมนู "รายการจอง"
  2. ดูรายการที่เป็น Pending → กดปุ่ม "จัดการ (ดินสอ)"
  3. ตรวจสอบรายละเอียด (วันที่, รถที่ User เลือกมา)
  4. Action:
    • หากรถว่าง: กดเปลี่ยนสถานะเป็น Approved
    • หากรถไม่ว่าง: สามารถ "เปลี่ยนรถ" ให้ User ได้ แล้วจึง Approve
    • หากไม่มีรถ: กด Rejected พร้อมระบุเหตุผล
  5. Assign Driver: เลือกคนขับให้งานนี้ (ระบบจะช่วยกรองคนขับที่ว่างให้)

Flow: การพิมพ์เอกสาร (Document)

  • เมื่อ Approve แล้ว จะมีปุ่มโผล่ขึ้นมา:
    • 📄 Word: ดาวน์โหลดแบบฟอร์ม "ใบขออนุญาตใช้รถ" (แบบ 3) ไปแก้ไขจัดหน้าได้
    • 📄 PDF: ดาวน์โหลดแบบพร้อมพิมพ์ (Format ราชการ: ตราครุฑ, เส้นประ, ฟอนต์ TH Sarabun)

C3. การเบิกน้ำมัน (Fuel Management)

  • ไปที่เมนู "จัดการน้ำมัน" (Fuel)
  • ตรวจสอบรายการที่ Driver ส่งเข้ามา
  • เปลี่ยนสถานะ:
    • In Progress: รับเรื่องแล้ว กำลังทำเบิก
    • Completed: เบิกจ่ายเสร็จสิ้น (ปิดงาน)

C4. รายงาน (Reports)

  • เมนู "รายงานการใช้รถ"
  • ตัวกรอง: เลือก เดือน/ปี หรือ เลือกเฉพาะทะเบียนรถ
  • ผลลัพธ์: ตารางแสดง วันที่, เวลา, ผู้ใช้, ภารกิจ, เลขไมล์ (ไป-กลับ), ระยะทางรวม, คนขับ
  • Export:
    • ปุ่ม excel (CSV): โหลดไปทำตารางต่อได้
    • ปุ่ม Print: พิมพ์หน้ารายงานสรุปพร้อมช่องเซ็นชื่อท้ายกระดาษ

C5. การจัดการข้อมูลหลัก (Master Data)

รถยนต์ (Vehicles)

  • เพิ่มรถใหม่: ใส่ทะเบียน, ยี่ห้อ, ประเภท (รถตู้/กระบะ), สีที่แสดงในปฏิทิน
  • สถานะรถ:
    • Active: พร้อมใช้
    • Repair: ส่งซ่อม (ระบบจะบล็อกไม่ให้จอง)
    • Inactive: งดใช้ชั่วคราว

พนักงานขับรถ (Drivers)

  • เพิ่มคนขับ: ใส่ชื่อ-สกุล, เบอร์โทร
  • สถานะการทำงาน:
    • Available: ว่าง
    • Busy: ติดภารกิจ
    • Off: ลา/หยุด
  • Connect Line: ดูสถานะว่าคนขับผูก Line แล้วหรือยัง ถ้ายังให้กดปุ่ม Copy Link ส่งให้เขา

ผู้ใช้งาน (Users)

  • สร้าง Username/Password ให้เจ้าหน้าที่
  • Role: กำหนดว่าเป็น USER หรือ ADMIN
  • Reset Password: ใช้เมื่อ User ลืมรหัสผ่าน

ภาคผนวก: สัญลักษณ์และสถานะระบบ

สถานะ (Status) สี (Color) ความหมาย
Pending สีเหลือง รอการอนุมัติจาก Admin
Approved สีเขียว อนุมัติแล้ว (มีรถและคนขับ)
Completed สีฟ้า ภารกิจเสร็จสิ้น (บันทึกเลขไมล์แล้ว)
Rejected สีแดง คำขอถูกปฏิเสธ
Cancelled สีเทา ผู้ใช้ยกเลิกคำขอเอง

สถานะรถ/คนขับ ความหมาย
Active / Available พร้อมให้บริการ
Repair รถอยู่ระหว่างซ่อมบำรุง (ห้ามจอง)
Busy คนขับติดภารกิจอื่นซ้อนอยู่
